# Dhan Trading Dashboard

A self-hosted trading dashboard for the [DhanHQ](https://dhanhq.co/) platform. Single Docker image that runs a FastAPI dashboard and JupyterLab for market analysis.

## Quick Start

```bash
# 1. Clone and configure
git clone https://github.com/athcodes2005/dhan-docker.git
cd dhan-docker
python generate_env.py   # interactive credential setup

# 2. Run
docker compose up -d
```

Dashboard: `http://localhost:8000` | JupyterLab: `http://localhost:8888/lab/`

## Docker Run

```bash
docker run -d \
  --name dhan-trading \
  --restart unless-stopped \
  -p 8000:8000 \
  -p 8888:8888 \
  --env-file .env \
  -v config:/config \
  -v data:/app/data \
  -v notebooks:/app/notebooks \
  athcodes2005/dhan-docker:latest
```

## What's Inside

| Service | Description | Port |
|---------|-------------|------|
| **Dashboard** | Portfolio overview, token management, instrument search | `8000` |
| **JupyterLab** | Market data analysis with pandas, plotly, pandas-ta | `8888` |

## Dashboard Pages

- **Home** — Fund limits and balance metrics
- **Authentication** — Token status, IP whitelisting, Data API status, auto-renewal toggle
- **Account** — Holdings and positions with portfolio history tracking
- **Search** — Instrument search across NSE/BSE exchanges
- **Lab** — JupyterLab environment for data analysis notebooks

## Login

| Username | Role | Access |
|----------|------|--------|
| `admin` | Admin | Full access: generate tokens, manage settings |
| `guest` | Guest | View only: balances, holdings, positions, search |

Passwords are configured in `.env` (`ADMIN_PASSWORD`, `GUEST_PASSWORD`).

## Environment Variables

Create a `.env` file manually or use `python generate_env.py`:

| Variable | Required | Description |
|----------|----------|-------------|
| `DHAN_CLIENT_ID` | Yes | Dhan client ID |
| `DHAN_API_KEY` | Yes | Dhan API key |
| `DHAN_API_SECRET` | Yes | Dhan API secret |
| `DHAN_PIN` | Yes | 6-digit trading PIN |
| `DHAN_TOTP_SEED` | Yes | TOTP seed for 2FA |
| `SECRET_KEY` | Yes | Session signing key (auto-generated by `generate_env.py`) |
| `ADMIN_PASSWORD` | Yes | Dashboard admin password |
| `GUEST_PASSWORD` | Yes | Dashboard guest password |
| `TZ` | No | Timezone (default: `Asia/Kolkata`) |

## Volumes

| Path | Purpose |
|------|---------|
| `/config` | `config.json` — access tokens, static IP, settings |
| `/app/data` | Instruments database, portfolio history |
| `/app/notebooks` | JupyterLab notebooks |

## Reverse Proxy

`docker-compose.yml` binds both ports to `127.0.0.1` — the container is **not**
reachable from outside the host. Put a reverse proxy in front to serve traffic.

JupyterLab is a code-execution shell with full access to your Dhan token.
**Never expose port 8888 publicly.** The dashboard provides
`GET /api/auth-check` (returns 200 for admin sessions, 401/403 otherwise) for
use as a reverse-proxy subrequest gate on `/lab/*`.

### Caddy example

```caddyfile
your-domain.example.com {
    encode zstd gzip
    header {
        Strict-Transport-Security "max-age=63072000; includeSubDomains; preload"
        X-Content-Type-Options "nosniff"
        Referrer-Policy "strict-origin-when-cross-origin"
        -Server
    }

    @lab path /lab/*
    handle @lab {
        forward_auth localhost:8000 {
            uri /api/auth-check
            copy_headers Cookie
        }
        reverse_proxy localhost:8888
    }

    handle {
        reverse_proxy localhost:8000
    }
}
```

The proxy and DNS/TLS setup are configured separately from this project.
